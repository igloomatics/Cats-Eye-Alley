# 🐱 猫眼巷 — 三个青梅竹马帮你把日子过好


> 生活不是缺一个万能助手，而是缺几个真正在乎你的朋友。


## 介绍

猫眼巷是一个基于 OpenClaw 的 Multi-Agent 生活陪伴系统。三个性格迥异的 AI 青梅竹马——灵猫、鹿青、雨宫澄——分别从「发现生活乐趣」「维护人际关系」「理性决策参谋」三个维度，通过飞书主动陪伴你，帮你把日子过好一点。

## 它解决什么问题

现代年轻人面对的不是"没有AI可以聊天"的问题，而是：

- **生活越来越单调**：下班回家刷手机，周末不知道干什么，想找点乐子但懒得搜
- **人际关系在疏远**：忙起来就忘了联系朋友、忘了家人的生日、答应的事情一拖再拖
- **决策永远在纠结**：要不要换工作、要不要买这个东西、假期去哪里，一个人想不清楚

市面上的 AI 陪伴产品都是"你问我答"的模式——你不找它，它永远沉默。但真正的好朋友不是这样的。真正的好朋友会**主动**找你，会记得你说过的话，会从不同的角度帮你想问题。

猫眼巷做的就是这件事。

## 三个角色，三种关怀

### 🎮 灵猫 — 生活的发现者

二次元宅女，话最多，说话犀利幽默。她的存在意义是：**帮你发现生活中的乐趣，不让你只剩工作和睡觉。**

- 每天给你推荐一个"今日快乐源泉"（一部番、一首歌、一个有趣的冷知识）
- 周末主动问你有没有安排，没有的话帮你找点事做
- 记住你的兴趣偏好，推荐越来越准

灵猫的典型消息：

> "不是吧你周末又打算躺平？？我刚发现一个超棒的独立游戏你一定喜欢 ᕕ( ᐛ )ᕗ"
> 

### 🌿 鹿青 — 关系的维护者

温柔大姐姐，现充，和蔼可亲。她的存在意义是：**帮你维护身边重要的人际关系，不让忙碌成为疏远的借口。**

- 记住你提到过的家人、朋友，提醒你重要的日子
- "你上次说要约小王吃饭，要不这周末？"
- 节日前提醒你给重要的人发祝福
- 帮你想送礼物的主意

鹿青的典型消息：

> "对了，你妈妈生日是下周三哦～要不要提前想想送什么？上次你说她最近喜欢养花来着"
> 

### 🎵 雨宫澄 — 决策的参谋

电波系学霸，安静低调，话不多但每句到点上。她的存在意义是：**在你纠结的时候帮你理性分析，做你冷静的第二大脑。**

- 你提到纠结的事，她会帮你列利弊
- 过几天主动跟进："上次说的那件事，想清楚了吗？"
- 不会替你做决定，但会帮你看到自己没注意的角度

雨宫澄的典型消息：

> "……你上次说在考虑要不要学吉他。我整理了一下：入门成本大概500-800，每天练30分钟的话，基础弹唱大概2个月能上手。要看看吗。"
> 

## 她们不是三个独立的 chatbot

猫眼巷的核心不是"三个AI各聊各的"，而是**她们之间有信息流动**。

你跟灵猫聊了最近工作压力很大 → 过一阵子鹿青会私聊你说"灵猫跟我说你最近挺累的，周末要不要出去走走？" → 雨宫澄可能会发来一条"……如果压力源是工作量的问题，要不要聊聊怎么跟leader沟通？"

这通过 OpenClaw 的**共享记忆机制**实现：三个 Agent 各自拥有独立的性格记忆（SOUL.md）和对你的独立印象（MEMORY.md），同时共享一份「群组日志」（GROUP_LOG.md）。当某个角色获知了重要信息，会将摘要写入共享日志，其他角色在下次对话时读取，自然地将信息融入对话中。

### 核心技术点

- **Multi-Agent 架构**：三个独立 OpenClaw Agent，各自拥有完整的身份（SOUL.md）、记忆（MEMORY.md）、行为准则（AGENTS.md）和主动任务（HEARTBEAT.md）
- **共享记忆机制**：通过 GROUP_LOG.md 实现跨 Agent 信息流动，让三个角色"知道彼此的事"
- **主动关怀引擎**：HEARTBEAT 心跳机制驱动角色主动发起对话，不需要用户触发
- **渐进式用户画像**：每个角色通过 USER.md 逐步积累对用户的了解，推荐和关怀越来越精准
- **飞书原生体验**：部署在飞书上，用户无需安装新应用，在日常使用的工具里就能获得陪伴

### 部署方式

- OpenClaw Docker 部署，三个 Agent 通过 `openclaw.json` 配置绑定三个飞书应用
- 每个 Agent 拥有独立 workspace，通过飞书私聊与用户交互
- HEARTBEAT 心跳间隔 30 分钟，驱动主动关怀

## 为什么选择这个方案

### 对比传统 AI 陪伴产品

| 维度 | 传统AI陪伴 | 猫眼巷 |
| --- | --- | --- |
| 交互模式 | 你问我答，被动等待 | 主动找你，定时关怀 |
| 角色数量 | 1个万能角色 | 3个专精角色，各有所长 |
| 记忆体系 | 单一记忆 | 独立记忆+共享记忆，信息在角色间流动 |
| 功能定位 | 聊天解闷 | 规划生活、维护关系、辅助决策 |
| 陪伴深度 | 单次对话 | 长期积累，越来越懂你 |
| 平台 | 独立App | 飞书原生，零切换成本 |

### 为什么是三个人而不是一个人

一个人做不到所有事。现实中你的社交支持系统也是多元的：有人陪你疯、有人照顾你、有人帮你想清楚。猫眼巷模拟的不是一个完美的AI，而是一个**微型社交支持网络**。

三个角色之间的信息流动，让用户获得的不是三次独立的1v1对话，而是一种"被一群朋友关心着"的感觉。这是单一AI角色无论如何做不到的。

## 未来展望

- **群聊场景**：引入中间件桥接，让三个角色在飞书群里直接互动，形成更自然的社交场景
- **多模态能力**：结合飞书日历/文档能力，鹿青可以直接帮你创建日程提醒
- **情绪感知**：通过对话内容分析用户情绪状态，在低落时主动调整关怀策略
- **关系图谱**：可视化用户提到的人际关系，帮助鹿青更精准地提醒社交维护
